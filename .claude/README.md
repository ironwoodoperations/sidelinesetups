# .claude/ — per-repo agent + hook layer

This directory is the **tactical** layer Claude Code reads inside each repo.

What ships in the template (framework pieces — upgraded in place over time):

- `settings.json` — wires the PreToolUse + Stop hooks
- `hooks/` — `require-pr.sh`, `protect-files.sh`, `session-end.sh`
- `commands/` — `/office-hours`, `/investigate`, `/qa`, `/review`, `/ship`

What is **generated per project** (not shipped in the template — a placeholder until then):

- `agents/` — the project's agent team, generated by the Harness meta-skill
  (vendored at `vendor/harness/` in HQ) or copied from a close team in `harness-100`.
- `skills/` — the skills those agents use, also generated by Harness.

When you generate the agent team for a new project, also copy
[`../GIT_RULES_REF.md`](../GIT_RULES_REF.md) into this `.claude/` so the generated agents
inherit the Ironwood git convention, and set its MERGE MODE per the paying-customer rule.

See `docs/HARNESS_USAGE.md` in HQ for the full new-project bootstrap.
